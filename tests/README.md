# SegFlow4D Tests

## Running Tests

All commands assume the `segflow4d-py310` conda environment and are run from the repo root.

**CPU-only (fast, no GPU required — suitable for most development)**
```bash
conda run -n segflow4d-py310 pytest tests/ -m "not gpu and not requires_real_data" -v
```

**GPU tests (requires a CUDA device)**
```bash
conda run -n segflow4d-py310 pytest tests/ -m "gpu" -v
```

**Full suite including real-data E2E (requires GPU + fixture files)**
```bash
# Generate fixtures first (one-time setup)
conda run -n segflow4d-py310 python scripts/generate_real_fixtures.py

# Then run
conda run -n segflow4d-py310 pytest tests/ -v
```

**Machine-readable JSON output (for coding agents)**
```bash
conda run -n segflow4d-py310 pytest tests/ -m "not gpu and not requires_real_data" \
  --json-report --json-report-file=test-results.json -v
```

**Coverage report**
```bash
conda run -n segflow4d-py310 pytest tests/ -m "not gpu and not requires_real_data" \
  --cov=src/segflow4d --cov-report=html
```

## Markers

| Marker | Meaning |
|---|---|
| `gpu` | Requires a CUDA-capable GPU. Skipped automatically on CPU-only machines. |
| `requires_real_data` | Requires fixture files in `tests/fixtures/real_data/`. Generate with `scripts/generate_real_fixtures.py`. |
| `slow` | Reserved for long-running tests. |

## Structure

```
tests/
├── conftest.py               # Shared CPU-only fixtures (synthetic images, meshes, warp fields)
├── fixtures/
│   └── real_data/            # Committed cropped patient data and baseline metrics
│       ├── image_4d_crop.nii.gz       (generated by scripts/generate_real_fixtures.py)
│       ├── seg_ref_crop.nii.gz        (generated by scripts/generate_real_fixtures.py)
│       └── expected_metrics.json      (baseline Dice for regression guard)
│
├── unit/                     # Tests for individual classes and functions in isolation
│   ├── test_image_wrapper.py         deepcopy, getters
│   ├── test_tp_data.py               clear(), deepcopy()
│   ├── test_image_processing.py      create_reference_mask, create_high_res_mask
│   ├── test_validation_metrics.py    Dice, HD95, macro avg, edge cases
│   ├── test_mesh_warper.py           identity and constant-displacement warp (gpu)
│   └── test_async_writer.py          image/mesh file writes, shutdown flush
│
├── integration/              # Tests for multi-component interactions; registration is mocked
│   ├── test_tp_partition.py          TPPartition initialization, TP count, masks
│   ├── test_propagation_strategies.py  Sequential and Star strategy ordering and output
│   └── test_propagation_input_factory.py  Factory from disk, error handling
│
└── e2e/                      # Full pipeline tests (gpu)
    ├── test_pipeline_synthetic.py    Output files, shape correctness, Dice ≥ 0.75
    └── test_pipeline_real.py         Smoke test and Dice regression on real fixture data
```
